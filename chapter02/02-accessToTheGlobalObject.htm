access to the global object

from book JavaScript Patterns pdf version, page 31

if you need to access the global object without hard-coding the identifier window, you can do the following from any level of nested function scope

this way you can always get the global object, because inside functions that were invoked as functions this should always point to the global object.

This is actually no longer the case in ECMAScript 5 in strict mode.

var global = (function() {
	return this;
}());
console.log(global === window);

(function(global) {
	"use strict";
	// access the global object via 'global'
	console.log(global === window);
}(this));
